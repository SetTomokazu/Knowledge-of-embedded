システム状態管理タスク¶

システム状態を管理？なんでしなくちゃいけないのさ

目的¶

OS は Task(タスク) を動かすことが役割であり、 Task(タスク) 間の協調までは管理しない為、管理用のタスクが必要なのである。
システムの立ち上げ、実行中、立ち下げの状態を監視し、他の Task(タスク) の制御を行う。
立ち上げ時には全ての Task(タスク) の初期化完了を通知を受け取ることで把握し、
立ち下げ時には各 Task(タスク) に対し動作停止命令を発行する。
Task(タスク) 間の連携を行う際に、動作してはいけない時に他の Task(タスク) から動作命令が出たりするのを防ぐ。

担当範囲¶

本タスクが行っていることは大体以下のようなもの

  • 各タスクへシステム状態の通知

  • 各タスクの状態把握

  • 電源状態や異常の把握とそれに伴う制御
