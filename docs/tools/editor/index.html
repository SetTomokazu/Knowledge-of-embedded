テキストエディタ¶

組み込み開発ではソースコードを編集するエディタとしてIDEではなく主にテキストエディタが使用される。
これは開発するマイコンによって開発環境が異なる為である。
各マイコンには独自の開発環境が存在するが、主な用途はビルドと実機デバッグ用である。
この為、ソースコードの編集も可能ではあるのだが以下のような理由で使い勝手が悪すぎる。

  • そもそも動作が重い。

  • Grep検索に対応していない場合が多い。

  • 定義元へのジャンプは実機デバッグ中でしか使えないものがある。

  • 文字入力中に補完が効かない。

  • 苦労して動作を覚えても開発対象のマイコンが変われば環境も変わる。

このため、ソースコードの編集は使い慣れたテキストエディタで行う。

  • VS Code
