開発の流れ¶

開発はほぼ完全にウォーターフォール・モデルに則って行われる。
そして各工程でそれなりにきっちりとしたドキュメントを作成することになる。
今時どうしてこんなしっかりウォーターフォールをやるのかと言えば、最たる要因が納品先がそれらのドキュメントを成果物として要求してくるからである。
そして車載開発においてその納品先の最大手とは誰もがよく知るT社である。
  • 【RA】要求分析
  • 【PD】詳細設計
  • 【MK】コーディング
  • 【CT】単体試験
  • 【IT】結合試験
  • 【ST】システム試験
